# HabitFlow — Production-Ready Cross-Platform Habit Tracker

A full-featured, production-ready habit tracking app built with Flutter 3.24+, Firebase backend, and offline-first architecture. Available for Android and iOS.

## ✨ Features

- **User Authentication**: Email/password + Google Sign-In via Firebase Auth
- **Onboarding**: 4-slide onboarding for new users
- **Habit Management**: Create, edit, delete habits with rich configuration
  - 5 habit types: Read Book (pages), Exercise (mins), Run (km), Sleep (hours), Custom
  - Goal periods: Daily, Weekly, Monthly
  - Multi-day scheduling (Mon–Sun)
  - Time range (start/end time)
  - Reminder notifications with time picker
  - Location-based triggers (Google Maps)
- **Dashboard**: Stats, today's habits, progress circles, streak counters
- **Charts**: Weekly bar chart + Monthly line chart (fl_chart)
- **Completion Logging**: Log progress with value input or quick toggle
- **Habit History**: View completion log with date-by-date breakdown
- **Notifications**: Scheduled local notifications via flutter_local_notifications
- **Settings**: Light/Dark/System theme, CSV export, sign out
- **Offline-First**: Hive local storage + Firestore sync on reconnect
- **Firebase Analytics**: Usage tracking

## 🚀 Getting Started

### Prerequisites

- Flutter SDK 3.24+
- Dart 3.4+
- Firebase account (free tier)
- Android Studio / Xcode (for device/emulator builds)

### 1. Clone & Install Dependencies

```bash
git clone https://github.com/CursedOn3/habit-tracker.git
cd habit-tracker
flutter pub get
```

### 2. Firebase Setup

#### Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **Add project** → name it `habit-tracker`
3. Enable **Google Analytics** (optional)

#### Enable Firebase Services

In your Firebase project:

- **Authentication** → Enable **Email/Password** and **Google** sign-in methods
- **Firestore Database** → Create database in **test mode** (then set security rules)
- **Analytics** → Already enabled if chosen above

#### Configure Android

1. In Firebase Console → **Project Settings** → **Your apps** → **Add app** → **Android**
2. Package name: `com.habittracker.app`
3. Download `google-services.json`
4. Place it at: `android/app/google-services.json`

#### Configure iOS

1. In Firebase Console → **Project Settings** → **Your apps** → **Add app** → **iOS**
2. Bundle ID: `com.habittracker.app`
3. Download `GoogleService-Info.plist`
4. Place it at: `ios/Runner/GoogleService-Info.plist`

#### Generate Firebase Options with FlutterFire CLI

```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=YOUR_PROJECT_ID
```

This generates/updates `lib/firebase_options.dart` with your real credentials.

### 3. Google Maps API Key

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Enable **Maps SDK for Android** and **Maps SDK for iOS**
3. Create an API key
4. Replace `YOUR_GOOGLE_MAPS_API_KEY` in:
   - `android/app/src/main/AndroidManifest.xml`
   - `ios/Runner/AppDelegate.swift`
   - `lib/utils/constants.dart`

### 4. Run the App

```bash
# Run on Android emulator/device
flutter run

# Run on iOS simulator/device
flutter run -d ios
```

## 📦 Build for Release

### Android (APK / App Bundle)

#### Set Up Signing

Create `android/key.properties`:
```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=YOUR_KEY_ALIAS
storeFile=../keystore/release.jks
```

Generate keystore:
```bash
keytool -genkey -v -keystore android/keystore/release.jks \
  -alias YOUR_KEY_ALIAS -keyalg RSA -keysize 2048 -validity 10000
```

#### Build

```bash
# App Bundle (for Play Store)
flutter build appbundle --release

# APK (for direct install)
flutter build apk --release --split-per-abi
```

Output: `build/app/outputs/bundle/release/app-release.aab`

### iOS (IPA)

```bash
flutter build ios --release --no-codesign
```

Then open `ios/Runner.xcworkspace` in Xcode, configure signing certificates, and archive.

## 🏪 Play Store / App Store Deployment

### Google Play Store

1. Create app in [Play Console](https://play.google.com/console)
2. Complete store listing (description, screenshots, icon)
3. Upload `app-release.aab` to Internal Testing
4. Add testers → promote to Production

### Apple App Store

1. Create app in [App Store Connect](https://appstoreconnect.apple.com/)
2. In Xcode: Product → Archive → Distribute App → App Store Connect
3. Submit for review in App Store Connect

## 🔒 Firestore Security Rules

In Firebase Console → Firestore → Rules:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/habits/{habitId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```
